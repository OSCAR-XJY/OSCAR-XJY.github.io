Modular Exponentiation

Posted on 2018-04-13 | In Algorithm

Description

Modular exponentiation is widely used, especially in the field of public-key cryptography. A time-saving method to solve the modular exponentiation problem makes great contributions to the processes of both encryption and decryption. For simplicity, we use $b$ (the base), $e$ (the exponent), $m$ (the modulus) below. Now, Let us try to calculate: ${b^e}\bmod m$. $(e \ge 0)$

Straightforward Method

Our first test case is $b = 17$, $e = 3$, $m=5$, which means to calculate:

$${17^{3}}\bmod 5$$

Firstly:

$${17^3} = 4913$$

then:

$$4913\bmod 5=3$$

The time complexity of this method is ${\rm O}\left( e \right)$, which suggests it’s not practical to apply this way in strong cryptography. Actually, in strong cryptology, $b$ is at least 78 decimal digits (256 bits), and $e$ can be even a larger number. Since ${b^e}$may contain thousands digits, ${b^e}$cannot be stored in a integer variable even if we use $long$ $long$ $int$. Meanwhile, we may spend a life time, waiting for the final answer.

Memory-efficient Method

Given two integers $a$ and $b$, the following two equations are equivalent:
$$
\begin{equation}
\begin{split}
&c\bmod m = \left( {a \cdot b} \right)\bmod m\\
&c\bmod m = \left[ {\left( {a\bmod m} \right) \cdot \left( {b\bmod m} \right)} \right]\bmod m\\
\end{split}
\nonumber
\end{equation}
$$
we can apply this on ${b^e}\bmod m$:
$$
\begin{equation}
\begin{split}
&{b^1}\bmod m = b\bmod m\\
&{b^2}\,\bmod \,m = \left[ {\left( {b\,\bmod \,m} \right) \cdot \left( {b\,\bmod \,m} \right)} \right]\,\bmod \,m\\
&{b^3}\,\bmod \,m = \left\{ {\left( {b\,\bmod \,m} \right) \cdot \left[ {\,{b^2}\bmod \,m} \right]} \right\}\,\bmod \,m\\
&\cdots\\
\end{split}
\nonumber
\end{equation}
$$
For instance:
$$
\begin{equation}
\begin{split}
&{17^1}\bmod 5 = 2\\
&{17^2}\,\bmod 5 = \,\left[ {\left( {17\,\bmod \,5} \right) \cdot \left( {17\,\bmod \,5} \right)} \right]\,\bmod \,5 = \left( {2 \cdot 2} \right)\bmod 5 = 4\\
&{17^3}\,\bmod 5 = \,\left\{ {\left( {17\,\bmod \,5} \right) \cdot \left[ {17^2\,\bmod \,5} \right]} \right\}\,\bmod \,5 = \left( {2 \cdot 4} \right)\bmod 5 = 3\\
\end{split}
\nonumber
\end{equation}
$$

The time complexity of this method is also ${\rm O}\left( e \right)$. But with the same input, integers manipulated here are much smaller than those in straightforward algorithm. So the time spent on computation will reduce, and the memory occupation will significantly decline. As a result, much larger $b$ and $e$ can be accepted by using this method. However, this method still not rapid enough, especially when we introduce our new test case: $b=13$, $e=1000000000$, $m=1000000007$.

Unfold Method

By unfolding $b^e$, we can quickly make $e$ smaller and smaller, and that just what we need. But remember we should be very careful in dealing the even or odd condition of $e$.
$$
{b^e} =
\begin{cases}
{\left( { {b^2} } \right)^{\frac{e}{2} } }, & \text{if $e$ is even} \\
b \cdot {\left( { {b^2} } \right)^{\frac{ {e - 1} }{2} } }, & \text{if $e$ is odd}
\end{cases}
$$
We observed no matter $e​$ is even or odd, $e​$ can always be reduced in a logarithmic manner. And by this way, we will finally reach $e=0​$.

$$
{b^e}\bmod m=
\begin{cases}
{\left( { {b^2} } \right)^{\frac{e}{2} } }\bmod m, & \text{if e is even} \\
b \cdot \left( { {b^{e - 1} } } \right)\bmod m = \left\{ {\left( {b\bmod m} \right) \cdot \left[ { { {\left( { {b^2} } \right)}^{\frac{ {e - 1} }{2} } }\bmod m} \right]} \right\}\bmod m, & \text{if $e$ is odd}
\end{cases}
$$

In case ${b^2}$ causes overflow, we can easily prove: ${\left( x \right)^n}\bmod m = {\left( {x\bmod m} \right)^n}\bmod m$. So:
$$
{b^e}\bmod m=
\begin{cases}
{\left( { {b^2}\bmod m} \right)^{\frac{e}{2} } }\bmod m, & \text{if e is even} \\
\left\{ {\left( {b\,\bmod \,m} \right) \cdot \left[ { { {\left( { {b^2}\bmod m} \right)}^{\frac{ {e - 1} }{2} } }\,\bmod \,m} \right] } \right\}\,\bmod \,m, & \text{if $e$ is odd}
\end{cases}
$$

The time complexity of this method is ${\rm O}\left( {\log e} \right)$. Which means this method is far rapider than all the methods we discussed above. Using it and without any delay, we can immediately find the answer for our test case $${13^{1000000000}}\bmod 1000000007 = {\rm{94858115}}$$

Right-to-left Binary Method

In mathematics and computer programming, exponentiation by squaring (binary exponentiation) is a general method for fast computation of large positive integer powers of a number. When $e>0$, we convert $e$ to its binary notation, with a length of $n$ bits, and with the value $0$ or $1$ for ${a_i}\left( {0 \le i \le n - 1} \right)$, which depended by the corresponding digit from ${\left( e \right)_2}$, then $e$ can be expressed as:

$$e = \sum\limits_{i = 0}^{n - 1} { {a_i} } {2^i} = {a_0}{2^0} + {a_1}{2^1} + {a_2}{2^2} + \cdots + {a_{n - 1} }{2^{n - 1} },n =\left\lfloor { { {\log }_2}(e) + 1} \right\rfloor$$

$b{}^e​$ can be expressed as:

$${b^e} = {b^{\left( {\sum\nolimits_{i = 0}^{n - 1} { {a_i}{2^i} } } \right)} } = \prod\limits_{i = 0}^{n - 1} { { {\left( { {b^{ {2^i} } } } \right)}^{ {a_i} } } } = {\left( { {b^{ {2^0} } } } \right)^{ {a_0} } } \cdot {\left( { {b^{ {2^1} } } } \right)^{ {a_1} } } \cdot {\left( { {b^{ {2^2} } } } \right)^{ {a_2} } } \cdots {\left( { {b^{ {2^{n - 1} } } } } \right)^{ {a_{n - 1} } } },n =\left\lfloor { { {\log }_2}(e) + 1} \right\rfloor$$

If we assume all the ${a_i}=1$, then we can use the transitivity between adjacent terms of ${b^e}\bmod m$:
$$
\begin{cases}
{\left( { {b^{ {2^i} } } } \right)^1}\bmod m = b\bmod m, & \text{$i=0$} \\
{\left( { {b^{ {2^i} } } } \right)^1}\bmod m = \left[ { { {\left( { {b^{ {2^{i - 1} } } } } \right)}^1}\bmod m \cdot { {\left( { {b^{ {2^{i - 1} } } } } \right)}^1}\bmod m} \right]\bmod m, & \text{$1 \le i \le n - 1$}
\end{cases}
$$
Notice we only calculate those truly ${a_i}=1​$ terms into result, other ${a_i}=0​$ terms we just use them, but not multiply them into result.
